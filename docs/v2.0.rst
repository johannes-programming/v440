Overview
--------

The project ``v440`` is a PyPI project
that provides mutable version objects
that allow flexible manipulation of version identifiers
following the ``PEP 440`` standard.

Installation
------------

To install ``v440``, you can use ``pip``.
Open your terminal and run:

.. code-block:: shell

    pip install v440

Typing
------

The typing conforms (not strictly) to ``mypy``.

Features
--------

Inherited features are only explained again
if there is further noteworthy information.

``v440.abc``
~~~~~~~~~~~~

``class v440.abc.CoreABC.CoreABC(string: object)``
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^

This abstract base class is the ancestor of all ``core`` classes.
It establishes some standards for its decendents.

All properties have always a getter, never a deleter, and sometimes a setter.
Properties without a setter are explained under ``NestedABC``.
The properties with setter are of an immutable builtin type.
The setter allows replacing the value with another.
If the setter receives an invalid value then a ``VersionError`` is raised
and the instance is left unchanged.

``__bool__() -> bool``
""""""""""""""""""""""

This magic method implements ``bool(self)``.
It is defined as equivalent to ``self == type(self)()``.

``__format__(format_spec: object) -> str``
""""""""""""""""""""""""""""""""""""""""""

The instances of each class can be formatted.
``format_spec`` is always either valid or invalid for a certain class.
The empty string is valid for all classes and provides the standardized string.
The ``format`` function always returns a ``PEP 440``-conforming string notation of the formatted object.
Each possible ``PEP 440``-conforming notation of an object has at least one valid ``format_spec``
that allows for its recreation in formatting.

``__eq__(other: object) -> Any``
""""""""""""""""""""""""""""""""

This abstract magic method will be overwritten to implement ``self == other``.
If ``type(self) is type(other)``,
the overwrites will always ``return self.string == other.string``.

``__ge__(other: Any) -> Any``
"""""""""""""""""""""""""""""

This abstract magic method will be overwritten to implement ``self >= other``.
See ``__le__``.

``__gt__(other: Any) -> Any``
"""""""""""""""""""""""""""""

This abstract magic method will be overwritten to implement ``self > other``.
See ``__le__``.

``__hash__ = None``
"""""""""""""""""""

This ``class`` and all its decendents are unhashable.

``__le__(other: Any) -> Any``
"""""""""""""""""""""""""""""

This abstract magic method will be overwritten to implement ``self <= other``.
The overwrites should always establish at the very least a partial order.
Indeed, each ``core`` class is ordered in accordance with ``PEP 440``.
Therefore, each class establishes a total order of all of its possible instances -
with the sole exception of ``Pre("")``
which is neither greater than nor lesser than any instances of ``Pre``.

``__lt__(other: Any) -> Any``
"""""""""""""""""""""""""""""

This abstract magic method will should overwritten to implement ``self < other``.
See ``__le__``.

``__ne__(other: object) -> Any``
""""""""""""""""""""""""""""""""

This abstract magic method will should overwritten to implement ``self != other``.
See ``__eq__``.

``__repr__() -> str``
"""""""""""""""""""""

``__subclasshook__(cls: type[Self], other: type, /) -> types.NotImplemented | bool``
""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""

This magic ``classmethod`` always returns ``NotImplemented``.

``__str__() -> str``
""""""""""""""""""""

This magic method implements ``str(self)``.
It returns ``self.string``.

``copy() -> Self``
""""""""""""""""""

This method returns a copy of ``self``.

``deformat(*strings: str) -> str``
""""""""""""""""""""""""""""""""""

This ``classmethod`` takes ``PEP 440``-conforming strings and returns the shorted possible ``format_spec`` that can recreate all of them in formatting.
The ``classmethod`` raises ``VersionError`` iff an argument is not actually ``PEP 440``-conforming or no common ``format_spec`` exists.

``packaging: Any``
""""""""""""""""""

This ``property`` connects ``v440`` with the ``packaging`` project.
Its getter converts ``self`` to an appropriate value for the ``packaging`` project.
Its setter takes such a value and configures ``self`` to match.

``string: str``
"""""""""""""""

This ``property`` represents ``self`` as a string.
``self.string`` returns the same value as ``str(self)``.
Setting this ``property`` allows to change the object in question to the state represented by the provided value.
The ``string`` property accepts any value iff that value provides a ``PEP 440``-conforming string.
It (and therefore also calling ``str``) always returns a standardized ``PEP 440``-conforming string. Two objects of the same type are equal iff their ``string`` properties are equal.

``v440.core``
~~~~~~~~~~~~~

This subpackage contains the principle classes of the project.
Each of these classes has a dedicated module inside ``core``.
Each class can be accessed alternatively in a nested fashion,
e.g. ``v440.core.Base.Base is v440.Version.Public.Base``.

``class v440.core.Base.Base(string: object = "0")``
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^

This ``class`` represents the base part of a public version identifier.

``epoch: int``
""""""""""""""

This ``property`` represents the epoch.

``packaging: str``
""""""""""""""""""

This ``property`` represents the equivalent ``packaging.version.Version.base_version`` value.

It is defined as an alias of ``string``.

``release: Release``
""""""""""""""""""""

See ``v440.core.Release.Release``.

``class v440.core.Dev.Dev(string: object = "")``
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^

This ``class`` represents the developmental release information.

``lit: str``
""""""""""""

This ``property`` represents the literal part of the information.
It can take the values ``""`` (with the numeral ``0``) and ``"dev"`` (with any non-negative numeral).

``num: int``
""""""""""""

This ``property`` represents the numeral part of the information.
It can only take non-negative values and is restricted to ``0`` if ``lit`` is ``""``.

``packaging: Optional[int]``
""""""""""""""""""""""""""""

This ``property`` represents the equivalent ``packaging.version.Version.dev`` value.

``class v440.core.Local.Local(string: object = "")``
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^

This list-like class represents the local part of version identifiers.
It inherits from ``v440.abc.ListABC.ListABC[int | str]``.

``packaging: Optional[str]``
""""""""""""""""""""""""""""

This ``property`` represents the equivalent ``packaging.version.Version.local`` value.

``lit: str``
""""""""""""

``num: int``
""""""""""""

This ``property`` represents the numeral part of the information. It can only take non-negative values and is restricted to ``0`` if ``lit`` is ``""``.

``packaging: Optional[int]``
""""""""""""""""""""""""""""

This ``property`` represents the equivalent ``packaging.version.Version.post`` value.

``class v440.core.Pre.Pre(string: object = "")``
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^

This ``class`` represents the pre-release information.

``lit: str``
""""""""""""

This ``property`` represents the literal part of the information.
It can take the values ``""``, ``"a"``, ``"b"``, and ``"rc"``. The value ``""`` demands the numeral ``0``. It signifies that we are not dealing with a pre-release. All other values tolerate any non-negative numeral. Each of them represents another phase of the pre-release process: alpha-releases, beta-releases, and previews.

``num: int``
""""""""""""

This ``property`` represents the numeral part of the information.
It can only take non-negative values and is restricted to ``0`` if ``lit`` is ``""``.

``packaging: Optional[tuple[str, int]]``
""""""""""""""""""""""""""""""""""""""""

This ``property`` represents the equivalent ``packaging.version.Version.pre`` value.
